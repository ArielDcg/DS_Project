Breve guía para instalar SFML (Simple and Fast Multimedia Library)

Requisitos:
- Compilador C++ (g++, clang, MSVC) y CMake.
- Conexión a Internet para descargar paquetes.

1) Ubuntu / Debian (rápido):
sudo apt update
sudo apt install libsfml-dev

Compilación con CMake:
mkdir build && cd build
cmake ..
cmake --build .

2) macOS (Homebrew):
brew update
brew install sfml

Compilación con CMake como arriba.

3) Windows (MSVC o MinGW):

Opción A — vcpkg (recomendado):
- Instala vcpkg (https://github.com/microsoft/vcpkg) y añade al PATH.
- Instala SFML:
  vcpkg install sfml:x64-windows
- Genera CMake usando el toolchain de vcpkg:
  cmake -B build -S . -DCMAKE_TOOLCHAIN_FILE=[RUTA_A_VCPKG]/scripts/buildsystems/vcpkg.cmake
  cmake --build build

Opción B — descarga manual:
- Descarga la versión apropiada en https://www.sfml-dev.org/download.php (elige MSVC/MinGW y arquitectura).
- Extrae en una carpeta y en CMake:
  cmake -B build -S . -DSFML_DIR="C:/ruta/a/SFML/cmake/"
  cmake --build build

4) MinGW (Windows):
- Asegúrate de usar la variante de SFML que coincida con tu MinGW (posix/seh) y la arquitectura (32/64-bit).
- Linkea las librerías estáticas o dinámicas según prefieras.

5) Ejemplo mínimo (CMakeLists.txt):
find_package(SFML 2.5 COMPONENTS graphics window system REQUIRED)

add_executable(mi_app src/main.cpp)
target_link_libraries(mi_app PRIVATE sfml::graphics sfml::window sfml::system)

6) Problemas comunes:
- Si CMake no encuentra SFML, especifica SFML_DIR apuntando al directorio cmake de SFML o instala mediante vcpkg.
- En Windows, usa la versión de SFML que coincide con tu compilador y arquitectura.
- Consulta la documentación oficial: https://www.sfml-dev.org/
